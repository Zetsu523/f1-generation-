# Cahier des charges — Site “F1 Knowledge Hub”

## 1. Présentation du projet
Le projet consiste à créer un site web éducatif consacré à la Formule 1. Le but est de permettre aux visiteurs de tout comprendre sur la F1 : son histoire, l'évolution des moteurs, les composants techniques d'une monoplace, les écuries, les voitures mythiques et le vocabulaire spécifique.

Le site sera développé en JavaScript avec React. Il sera entièrement codé avec Codex et hébergé sur GitHub Pages.

## 2. Objectifs
- Créer un site clair et pédagogique sur la F1.
- Expliquer les composants d'une monoplace simplement.
- Montrer l'évolution technique de la F1 au fil des années.
- Proposer des frises chronologiques et tableaux comparatifs.
- Offrir une navigation fluide sur ordinateur, tablette et mobile.
- Publier le site gratuitement via GitHub Pages.

## 3. Public cible
- Débutants qui veulent comprendre la F1.
- Passionnés souhaitant retrouver des informations techniques.
- Étudiants ou curieux intéressés par l'histoire automobile.
- Personnes voulant comprendre les différences entre les générations de moteurs.

## 4. Périmètre fonctionnel
### Inclus dans la V1
- Page d'accueil.
- Page histoire de la F1.
- Page évolution des moteurs.
- Page composants d'une F1.
- Page voitures mythiques.
- Page écuries.
- Page glossaire.
- Recherche simple dans le glossaire.
- Données stockées en JSON.
- Site responsive.
- Déploiement GitHub Pages.

### Non inclus dans la V1
- Backend.
- Compte utilisateur.
- Base de données distante.
- Espace administrateur.
- API payante.
- Commentaires utilisateurs.

## 5. Contraintes
- Le site doit rester statique.
- Le projet doit fonctionner avec GitHub Pages.
- Le code doit rester simple, propre et maintenable.
- Les informations historiques précises doivent être sourcées.
- Le site doit être compatible mobile.

## 6. Technologies prévues
- React
- JavaScript
- Vite
- React Router
- CSS classique ou CSS Modules
- JSON pour les données
- GitHub Pages pour l'hébergement

## 7. Identité visuelle
Style recommandé : moderne, sombre, sportif et premium.

Couleurs possibles :
- Noir / gris carbone pour le fond.
- Rouge pour l'accent F1.
- Blanc pour le texte principal.
- Gris clair pour les textes secondaires.

## 8. Critères de réussite
- Le site se lance en local avec `npm run dev`.
- Le site se compile avec `npm run build`.
- Toutes les pages principales sont accessibles.
- Le site est lisible sur mobile.
- Le site est publié sur GitHub Pages.
- Les données sont faciles à compléter.
