# Backlog et user stories

## Sprint 1 — Initialisation
### US01 — Créer le projet React
En tant que développeur, je veux initialiser un projet React avec Vite afin d'avoir une base propre.

Critères d'acceptation :
- Le projet se lance avec `npm run dev`.
- Le projet compile avec `npm run build`.
- L'arborescence de base est créée.

### US02 — Ajouter le layout global
En tant qu'utilisateur, je veux voir un header et un footer sur toutes les pages afin de naviguer facilement.

Critères d'acceptation :
- Header visible sur toutes les pages.
- Footer visible sur toutes les pages.
- Navigation fonctionnelle.

## Sprint 2 — Pages principales
### US03 — Créer la page Accueil
En tant que visiteur, je veux comprendre immédiatement le but du site.

### US04 — Créer la page Histoire
En tant que visiteur, je veux découvrir les grandes périodes de la F1.

### US05 — Créer la page Moteurs
En tant que visiteur, je veux comprendre l'évolution des moteurs de F1.

### US06 — Créer la page Composants
En tant que visiteur, je veux comprendre à quoi servent les pièces d'une F1.

## Sprint 3 — Données et composants réutilisables
### US07 — Créer les fichiers JSON
En tant que développeur, je veux stocker les données dans des fichiers JSON pour pouvoir les compléter facilement.

### US08 — Créer un composant Card
En tant que développeur, je veux un composant carte réutilisable.

### US09 — Créer une Timeline
En tant que visiteur, je veux visualiser l'évolution de la F1 dans le temps.

## Sprint 4 — Recherche et comparaison
### US10 — Ajouter une recherche dans le glossaire
En tant que visiteur, je veux rechercher rapidement un terme technique.

### US11 — Ajouter un tableau comparatif des moteurs
En tant que visiteur, je veux comparer les grandes générations de moteurs.

## Sprint 5 — Responsive et déploiement
### US12 — Optimiser le responsive
En tant qu'utilisateur mobile, je veux consulter le site confortablement sur téléphone.

### US13 — Déployer sur GitHub Pages
En tant que propriétaire du site, je veux rendre le site accessible publiquement.
